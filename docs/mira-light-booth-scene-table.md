# Mira Light 展位场景表

## 文档目的

注意：

当前仓库里的动作实现与排练审计，已经优先以 [`Mira Light 展位交互方案2.pdf`](../docs/Mira%20Light%20展位交互方案2.pdf) 为准。

本文件保留为场景总表和导演层阅读材料，但其中部分动作描述仍带有早期稿件口径，不应直接视为当前代码真值。

当前实现状态、已修问题和剩余缺口请优先对照：

- [`docs/mira-light-pdf2-implementation-audit.md`](../docs/mira-light-pdf2-implementation-audit.md)
- [`docs/mira-light-pdf2-engineering-handoff.md`](../docs/mira-light-pdf2-engineering-handoff.md)

这份文档把 [`Mira Light 展位交互方案.pdf`](../docs/Mira%20Light%20展位交互方案.pdf) 中的自然语言场景，整理成可以直接用于：

- 展位排练
- 主持人口播
- 控制脚本编排
- 设备验收
- 故障回退

每个场景都统一拆成：

- 场景目标
- 触发条件
- 主持人口播
- 动作序列
- 灯光设计
- 依赖能力
- 触发方式
- 失败回退

## 使用方法

这份场景表最适合搭配三类系统一起用：

1. 主持人手卡  
2. 展位控制台  
3. 单片机导演脚本

当前更符合实际的触发方式只有两种：

1. `OpenClaw` 触发  
思路是让 `OpenClaw` 最终调用电脑终端命令，执行 [`scripts/booth_controller.py`](../scripts/booth_controller.py)。

2. 电脑终端直接触发  
例如：

```bash
python3 scripts/booth_controller.py --base-url http://172.20.10.3 wake_up
python3 scripts/booth_controller.py --base-url http://172.20.10.3 celebrate
python3 scripts/booth_controller.py --base-url http://172.20.10.3 farewell
```

如果后面真的做网页控制台、物理按钮或快捷键，那应该视作后续增强，而不是当前默认前提。

## 场景 1：起床

| 项目 | 内容 |
| --- | --- |
| 场景名 | 起床 |
| 目标情绪 | 苏醒、欢迎、轻微笨拙感 |
| 观众感受 | 它不是机械启动，而是像一个刚醒的小动物 |
| 触发条件 | 检测到人靠近约 `1.5m`，或主持人决定开始开场场景 |
| 主持人口播 | “当 Mira 感觉到有人靠近，它不会立刻机械转头，而是像刚睡醒一样慢慢睁眼、抖一抖、伸个懒腰。” |
| 动作序列 | 微光亮起 -> 灯臂缓缓升起 -> 中途抖两下 -> 灯头抬起 -> `stretch` -> 回到正常姿态 |
| 灯光设计 | 先暖色微光，再过渡到柔和常亮 |
| 建议代码原语 | `light_dim_warm`, `arm_raise_slow`, `micro_shiver`, `act_stretch`, `pose_normal` |
| 依赖能力 | 人体接近检测；如果还没接传感器，则由 Claw 或终端直接触发 |
| 触发方式 | `OpenClaw`：运行 `python3 scripts/booth_controller.py --base-url http://172.20.10.3 wake_up`；终端：直接运行同一命令 |
| 失败回退 | 若传感器失效，直接用终端命令或 Claw 命令触发固定动作序列 |
| 验收标准 | 动作总时长约 `4~6s`；视觉效果是“醒来”，不是“突然弹起” |

## 场景 2：好奇你是谁

| 项目 | 内容 |
| --- | --- |
| 场景名 | 好奇你是谁 |
| 目标情绪 | 好奇、怯生生、礼貌 |
| 观众感受 | 它会观察你、试探你，而不是冷冰冰对准你 |
| 触发条件 | 评委到桌前，处于正前方区域；或主持人决定进入“好奇观察”段落 |
| 主持人口播 | “Mira 会先试探着看你，而不是直接盯着你。这种停顿和歪头，会让人立刻觉得它是活的。” |
| 动作序列 | 朝目标半转 -> 停顿 -> 继续转向 -> 左歪头 -> 停 `2s` -> 轻点头 |
| 灯光设计 | 柔和暖白，不抢动作戏份 |
| 建议代码原语 | `pose_curious_half_turn`, `turn_to_target_full`, `pose_head_tilt_left`, `act_nod` |
| 依赖能力 | 目标方向估计，或预设左/中/右方向 |
| 触发方式 | `OpenClaw`：运行 `python3 scripts/booth_controller.py --base-url http://172.20.10.3 curious_observe`；终端：直接运行同一命令 |
| 失败回退 | 如果目标定位失败，固定朝主持人预设方向执行 |
| 验收标准 | 必须有“半转 + 停顿 + 歪头 + 点头”四段节奏 |

| 场景名 | 摸一摸 |

| 建议代码原语 | `pose_extend`, `rub_motion`, `follow_target_once`, `pose_normal`, `light_warmest` |
| 依赖能力 | 手部位置检测、触摸传感器；如果没有，则由 Claw 或终端直接切场景 |
| 触发方式 | `OpenClaw`：运行 `python3 scripts/booth_controller.py --base-url http://172.20.10.3 touch_affection`；终端：直接运行同一命令 |
| 失败回退 | 如果检测不到手，直接演“前探 + 左右轻蹭”固定版本 |
| 验收标准 | 动作应小而亲密，不能像攻击或撞击 |

## 场景 5：发呆

| 观众感受 | 它不是只会等待命令，而像在自己想事情 |

| 灯光设计 | 保持当前工作光，或略降亮度 |
| 建议代码原语 | `look_random_direction`, `hold`, `snap_back_to_normal`, `sleepy_drop_and_bounce` |
| 依赖能力 | 计时器即可 |
| 触发方式 | `OpenClaw`：运行 `python3 scripts/booth_controller.py --base-url http://172.20.10.3 daydream`；终端：直接运行同一命令 |
| 失败回退 | 用版本 A 固定角度模拟，不依赖随机目标 |
| 验收标准 | 重点是“停住那几秒”，要有空白感 |

## 建议现场优先使用的 MVP 场景

如果现场时间和稳定性有限，建议只保留下面 6 个作为必演版本：

| 优先级 | 场景 | 原因 |
| --- | --- | --- |
| P0 | 起床 | 开场最抓人 |
| P0 | 好奇你是谁 | 最能体现“活物感” |
| P0 | 摸一摸 | 最有情感连接 |
| P0 | 追踪 | 最能证明“它看得见” |
| P0 | 跳舞模式 | 最有展位爆点 |
| P0 | 挥手 + 睡觉 | 演示闭环完整 |

其余场景可作为加分项或备选。

## 建议控制台字段

为了让这份场景表真正能用于展位执行，控制台最好至少显示：

- 当前场景名
- 当前状态
- 当前目标方向
- 当前动作是否播放中
- 当前灯光模式
- 当前 4 个关节角度
- 当前触发来源

这里的“当前触发来源”建议统一只记录两类：

- `openclaw`
- `terminal`
