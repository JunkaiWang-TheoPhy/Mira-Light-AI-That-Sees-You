# 实时语音常见问题排查

## 1. 脚本起不来

先运行：

```bash
scripts/mira-talk-open.sh --help
```

如果这条都起不来，再检查：

```bash
ls .venv/bin/python
```

## 2. 没有听到声音

先看是不是 dry-run：

```bash
scripts/mira-talk-open.sh --dry-run-audio
```

如果 dry-run 能看到 `speaker-hp-tts-play` 命令，说明对话链路基本是通的，问题更可能在音箱输出。

## 3. 说了一句但没有回复

现在系统会主动过滤这些内容：

- 空文本
- 重复垃圾转写
- 单字转写
- 低信息短片段

终端里常见表现是：

```text
[turn 001] skipped: short-low-information-transcript
```

或者：

```text
[turn 001] skipped: empty-transcript
```

## 4. 想先只测对话，不让灯动作

直接加：

```bash
--no-trigger
```

完整例子：

```bash
scripts/mira-talk-open.sh --no-trigger
```

## 5. 误触太多

改用手动开始、自动结束模式：

```bash
scripts/mira-talk-manual.sh
```

## 6. 回复感觉慢

当前已经默认启用低延迟 preset。

如果你想确认当前是不是低延迟模式，可以看启动文本里是否有：

```text
Latency preset: low
```

如果你故意切回标准模式，会更慢：

```bash
scripts/mira-talk-open.sh --latency-preset standard
```

## 7. 想看麦克风设备名

```bash
scripts/run_mira_realtime_voice_interaction.sh --list-inputs
```

## 8. 想切声音

女声：

```bash
scripts/mira-talk-open.sh --voice-mode gentle_sister
```

男声：

```bash
scripts/mira-talk-open.sh --voice-mode warm_gentleman
```

## 9. 想先回放现成音频

```bash
scripts/run_mira_realtime_voice_interaction.sh \
  --file runtime/realtime-voice-interaction/2026-04-09T13-06-20-940875/turn-001/input.wav \
  --once \
  --dry-run-audio \
  --no-trigger
```

## 10. 最小排查顺序

第一步：

```bash
scripts/run_mira_realtime_voice_interaction.sh --list-inputs
```

第二步：

```bash
scripts/mira-talk-open.sh --help
```

第三步：

```bash
scripts/mira-talk-open.sh --no-trigger
```

第四步：

```bash
scripts/mira-talk-manual.sh --no-trigger
```
